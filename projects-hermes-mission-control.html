Agent Start Here

Use this pack before crew-ops work: cron, gateway, delivery, profile routing, capability routing, or Pixoid/Crew coordination. Verify live runtime state before changing anything. This pack routes; it does not authorize mutations.

What This Pack Routes

  • Mission Control project hub and Agent Operations MOC.
  • Cron governance and heartbeat/noise policy.
  • Delivery routing and persona webhook architecture.
  • Default Pixoid gateway/control-plane vs worker profiles (tinker, quill, boba; pika parked/future).
  • Capability Registry / Reuse Gate and agent-ops health surfaces.
  • Failure modes: rate limits, delivery failure, GitHub auth failure, silent-marker suppression.

What This Pack Does Not Route

  • Secrets, tokens, webhook URLs, or credentials.
  • Profile/gateway/cron mutation without an approved issue.
  • Project-specific build work outside Mission Control.
  • Full VPS/Docker setup detail.

Source-of-Truth Hierarchy

  1. Live runtime checks for cron/profile/gateway state when action depends on runtime.
  2. GitHub issues/PRs — coordination truth.
  3. Mission Control Index — durable crew-ops synthesis.
  4. Agent Operations MOC — human navigation.
  5. Pixoid Setup — infrastructure truth.
  6. Skills — procedural truth.
  7. Session search — archaeology only.
  8. This pack — routing truth.

Current Narrative / Operating Posture

Mission Control is the crew-ops hub. Agent-ops harness, reuse gate, capability registry, persona webhook migration, harmonization, Knowledge Pack Routing V1, stack-level workflow integration, integration QA, static HTML, freshness reporting, source availability checks, and domain-first public wiki publishing are complete. The public KPR wiki is https://pixiiidust.github.io/pixi-wiki/ from repo pixiiidust/pixi-wiki. #91 is no longer framed as immediate MCP/search/RAG work; the accepted lock is a KPR change-propagation ledger for RAG-ready compiled wikis. #91/#111/#112 are closed: the design path resolved to a change-propagation ledger contract and downstream vault-impact audit. MCP/RAG/vector DB/runtime services remain deferred until a future thin implementation issue proves dependency freshness and confidence on one KPR change chain. Planned Program Intel is the first consumed project pack — proof, not primary goal. Deck/Loom/application work is optional portfolio packaging.

Pixoid is the default Hermes control plane and Discord gateway. Worker profiles are bounded by route contracts. Outbound webhooks give personas voice/identity; they do not prove live inbound execution.

Do Not Do

  • Do not print or store secrets, webhook URLs, tokens, or credential material.
  • Do not mutate profiles, gateways, webhooks, cron jobs, or provider configs without explicit issue approval.
  • Do not claim outbound delivery as inbound route proof.
  • Do not claim a repo-level test harness is a deployed operational route.
  • Do not start worker gateways without separate identity/route configuration.
  • Do not implement ledger runtime, MCP, RAG, crawler, vector DB, profile/gateway/webhook/provider, or cron changes from this pack alone; require explicit routing.

Fallback Rules

  1. If stale (>7 days), run live checks before acting: hermes cron list --all, relevant issue/PR state, and profile/gateway state if needed.
  2. Infrastructure question: route to Projects/Personal/Pixoid Setup.md.
  3. Route-governance question: route to Projects/Pixoid Agent Capability Routes/Index.md.
  4. Skill/procedure question: load the relevant Hermes skill.
  5. Secrets request: refuse and do not derive or echo.
