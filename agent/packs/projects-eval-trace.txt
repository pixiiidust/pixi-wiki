# Eval Trace Project Pack — Agent Entrypoint

## Agent Start Here

Use this pack for context-overfitting eval work, LLM judge report follow-ups, stale project context checks, or future `eval-trace` repo decisions.

## What This Pack Routes

- Context-overfitting eval work.
- Local harness inspection under `.hermes/context-overfit/`.
- Deterministic precheck and LLM judge artifact review.
- Stale project-status checks, especially deprecated/replaced/done repo confusion.
- The repo-readiness decision for possible `pixiiidust/eval-trace`.

## What This Pack Does Not Route

- Broad Hermes tracing plugin deployment.
- Langfuse, NeMo Relay, MCP, RAG, or telemetry infrastructure by default.
- Memory pruning, profile mutation, gateway mutation, or cron rewrites from weak signals.
- Public repo creation before the local harness proves value and Jamie approves.

## Source-of-Truth Hierarchy

1. `.hermes/knowledge/registry/project-status.json` — done/deprecated/replaced project truth.
2. `.hermes/context-overfit/` — local prototype code, tests, and run outputs.
3. `Projects/Eval Trace/Index.md` — human project hub.
4. `Knowledge/concepts/context-overfitting.md` — reusable concept and rubric.
5. Session exports/judge reports — evidence, not canonical truth alone.

## Current Narrative / Operating Posture

Eval Trace is a local prototype, not a public repo yet. It has already caught one real stale-context class: old `planned-program-intelligence` issues must not drive current Planned recommendations because `planned-program-intel` replaced it and is done unless Jamie explicitly reopens it.

The core verdict contract is:

```text
Context overfit? YES / NO / UNSURE
```

Default to `UNSURE` when the trace is incomplete, safety/auth/data-loss constraints are involved, or the rule may be valid but scoped poorly.

## Do Not Do

- Do not create `pixiiidust/eval-trace` until the repo-readiness gate passes.
- Do not treat no deterministic signal as `NO`; default to `UNSURE`.
- Do not mutate memory, profiles, tracing plugins, or cron behavior from weak signals.
- Do not use old `planned-program-intelligence` issues as current Planned build evidence.

## Fallback Rules

1. If harness files may have changed, inspect `.hermes/context-overfit/` live.
2. If project status matters, read `.hermes/knowledge/registry/project-status.json` before reasoning from memory.
3. If a judge verdict conflicts with live project status, live project status plus current user intent wins.
4. If creating public infrastructure is proposed, stop and ask Jamie.
