Agent Start Here

Use this pack after the root /llms.txt when working with the Obsidian Knowledge wiki. This pack routes agents to schema/index/log/source obligations. It is routing truth, not canonical truth.

What This Pack Routes

  • Knowledge/SCHEMA.md — operating contract and OKF-compatible canonical-note rules.
  • Knowledge/index.md — catalog of concepts, entities, comparisons, queries, and meta pages.
  • Knowledge/log.md — append-only maintenance trail.
  • Knowledge/AGENTS.md — short local editing instructions.
  • Knowledge/concepts/, entities/, comparisons/, queries/, _meta/, and raw/.
  • Boundaries between Obsidian Knowledge, local Hermes knowledge, memory/Honcho, project truth, and skills.

What This Pack Does Not Route

  • Live project implementation or issue state.
  • Local Hermes knowledge under ~/.hermes/knowledge/ except as a fallback/pointer.
  • Memory/Honcho internals or transcript archaeology.
  • Broad schema migrations or mass frontmatter rewrites.
  • Runtime Hermes profile, gateway, cron, provider, or webhook mutation.

Current Narrative / Operating Posture

Obsidian Knowledge is Jamie's human-facing durable knowledge wiki for reusable concepts, entities, comparisons, and substantial queries. It mirrors some local Hermes knowledge concepts when useful, but it is not the same layer as ~/.hermes/knowledge/ and not a replacement for memory/Honcho.

The agent-wikis concept preserves the broader pattern: maintained, scoped, source-backed knowledge surfaces for agents. Knowledge Pack Routing is the current Pixiedust implementation.

Do Not Do

  • Do not edit files under Knowledge/raw/ after ingest.
  • Do not create pages for passing mentions.
  • Do not put project execution state here when it belongs in GitHub/project hubs.
  • Do not put long concept essays in memory.
  • Do not create procedures here when they belong in skills.
  • Do not mass-migrate or rewrite schema/frontmatter from this pack alone.

Fallback Rules

  1. Query existing Knowledge: read Knowledge/index.md, then search Knowledge/.
  2. Creating/updating a Knowledge page: read SCHEMA.md, index.md, recent log.md, and search for existing pages first.
  3. After a content change: update the page, refresh index.md, append log.md, and update Knowledge MOC only if retrieval path changed.
  4. Project-specific fact: route to Projects domain and project/GitHub truth.
  5. Agent-operational reusable concept: check ~/.hermes/knowledge/llms.txt and local Hermes knowledge before duplicating.
