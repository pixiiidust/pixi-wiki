Agent Start Here

Use this pack when working on MOCs themselves or when deciding whether a human navigation surface needs maintenance. MOCs are human navigation surfaces. llms.txt files are agent retrieval contracts. Do not collapse one into the other.

What This Pack Routes

  • Which MOC to inspect for vault navigation.
  • How agents should maintain MOCs without treating them as canonical truth.
  • Boundaries between MOCs, root/project/knowledge packs, project hubs, and GitHub.
  • The current decision that Templates/llms.txt is deferred unless template-routing rules become distinct enough to justify it.

What This Pack Does Not Route

  • Current project implementation state.
  • Full Knowledge wiki content.
  • Template body content.
  • Asset/evidence handling under Attachments/.
  • Runtime automation, cron, MCP, RAG, crawler, search, vector DB, or public hosting.

Source-of-Truth Hierarchy

  1. Canonical domain sources — GitHub/project hubs/Knowledge schema/raw sources, depending on the topic.
  2. Domain packs — root, Projects, Knowledge, and project-specific llms.txt files.
  3. MOCs — human navigation and cross-link surfaces.
  4. This pack — routing truth for MOC maintenance boundaries.

Current Narrative / Operating Posture

The vault now has a small agent entrypoint mesh: root, Projects, Knowledge, project-specific packs, and this MOC domain pack. MOCs remain the human map layer. Agents should use MOCs to orient and discover, then verify against canonical domain sources before acting.

Templates currently have a compact Templates/README.md and do not need a separate Templates/llms.txt. Create one later only if template selection/maintenance becomes a recurring agent routing problem.

Do Not Do

  • Do not treat MOC text as proof of current project state.
  • Do not duplicate full project or Knowledge indexes inside this pack.
  • Do not create one pack per folder by default.
  • Do not create Templates/llms.txt just because the folder exists.
  • Do not perform broad vault reorganization from this pack.

Fallback Rules

  1. Need a project: root pack → Projects pack → project-specific pack or project Index.md → GitHub.
  2. Need a reusable concept: root pack → Knowledge pack → Knowledge index/schema → local Hermes knowledge if agent-operational.
  3. Need human navigation maintenance: this pack → relevant MOC → canonical domain source.
  4. Need a template: use Templates/README.md; stop and create a Templates pack only if distinct routing rules are necessary.
