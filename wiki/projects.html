Agent Start Here

Use this pack after the root /llms.txt when the task is project-related but before choosing a specific project hub. This pack routes agents through Jamie's project bounded contexts. It is routing truth, not canonical truth.

What This Pack Routes

  • Active project contexts.
  • Parked and historical project contexts.
  • Project-specific knowledge-pack coverage.
  • Which project surfaces are canonical for current state.
  • Proof-project drift guardrails.

What This Pack Does Not Route

  • Detailed code implementation steps.
  • Live repo branch/PR truth without GitHub verification.
  • Runtime Hermes profile/gateway/cron mutations.
  • Personal secrets, credentials, or private env details.

Source-of-Truth Hierarchy

  1. GitHub issues/PRs — coordination and implementation truth.
  2. Project PRDs/ADRs — requirements and architecture truth.
  3. Project Index.md notes — durable project synthesis and boundaries.

  6. Projects MOC — human project map.
  7. Skills — procedures.
  8. Session search/web — fallback/archaeology only.

Do Not Do

  • Do not implement parked/historical projects without explicit reopening.
  • Do not trust project status from this pack without checking the listed canonical sources.
  • Do not create a project-specific pack unless the project has distinct routing/freshness/ownership needs.
  • Do not create runtime automation from this pack.

Fallback Rules

  1. Project has llms.txt: load it, then its canonical sources.
  2. Project has no llms.txt: load Projects/<Project>/Index.md.
  3. Project has no index: use Projects MOC and Project Board.
  4. Current implementation state matters: verify GitHub issues/PRs and repo state live.
  5. Project is parked/historical: stop unless Jamie explicitly reopens it.
