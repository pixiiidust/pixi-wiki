Agent Start Here

Before project work, load this file to choose the right source. Then load the relevant project knowledge pack or project Index.md. This file is routing truth, not canonical truth.

What This Pack Routes

  • Vault-wide source-of-truth hierarchy.
  • Active project priority and next entrypoint.
  • Where to find project hubs, MOCs, and Knowledge wiki surfaces.
  • The distinction between human navigation (MOCs/indexes) and agent retrieval contracts (llms.txt).
  • Crew roles: Pixoid routes/reviews; Quill maintains packs; Boba contributes sources; Tinker contributes implementation facts.

What This Pack Does Not Route

  • Detailed project issue trees or implementation instructions.
  • Live Hermes gateway/profile/cron mutation.
  • Personal secrets or credentials.
  • External web research except as fallback.

Source-of-Truth Hierarchy

  1. GitHub issues/PRs — coordination truth.
  2. PRDs/ADRs — requirements and architecture truth.
  3. Obsidian project notes — human synthesis and durable project context.
  4. Hermes local knowledge — reusable concepts and agent-operational facts.
  5. Skills — procedural truth.
  6. Memory / USER — stable Jamie/environment preferences only.
  7. Session search — conversation archaeology, never canonical alone.
  8. Web / public Agent Wikis — external/current fallback.
  9. Knowledge packs — routing truth: maps to the right source above.

Do Not Do

  • Do not treat MOCs as agent retrieval contracts; they are human navigation.
  • Do not treat knowledge packs as canonical project truth.
  • Do not use session search as the first source for current project state.
  • Do not conflate planned-program-intel (active) with planned-program-intelligence (historical).
  • Do not create runtime automation, MCP, HTML publishing, or freshness cron as part of V1.

Fallback Rules
